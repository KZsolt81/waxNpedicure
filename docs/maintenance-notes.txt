UPDATING CONTENT
----------------

All business-specific content lives in /content/*.json files.
Edit these files to update prices, services, areas, FAQ, testimonials, and offers
WITHOUT touching any React/TypeScript code.

  content/business-info.json  — name, branding, hours, policies
  content/services.json       — service categories and descriptions
  content/pricing.json        — prices, durations, travel fees
  content/bundles.json        — bundle combinations and savings
  content/faq.json            — FAQ questions and answers (categorised)
  content/testimonials.json   — client reviews
  content/service-areas.json  — covered postcodes and travel info
  content/offers.json         — promotions (set active: true/false to show/hide)

BOOKING FLOW (SQUARE APPOINTMENTS)
----------------------------------

Booking is now handled through Square Appointments.
Primary booking URL is centralized in:

  src/lib/booking.ts
  SQUARE_BOOKING_URL = process.env.NEXT_PUBLIC_SQUARE_BOOKING_URL || "PASTE_SQUARE_BOOKING_URL_HERE"

Environment variable:

  .env.local.example
  NEXT_PUBLIC_SQUARE_BOOKING_URL=PASTE_SQUARE_BOOKING_URL_HERE

WhatsApp chat URL is centralized in:

  src/lib/booking.ts
  WHATSAPP_CHAT_URL = process.env.NEXT_PUBLIC_WHATSAPP_CHAT_URL || "PASTE_WHATSAPP_CHAT_URL_HERE"

Environment variable:

  .env.local.example
  NEXT_PUBLIC_WHATSAPP_CHAT_URL=PASTE_WHATSAPP_CHAT_URL_HERE

Current policy implemented across site content:
- No upfront payment at booking
- Card details may be required to secure appointment
- Less than 24 hours cancellation or no-show: 50% of selected treatment price may be charged
- Remaining balance is paid after treatment

Square embed/button placeholder is on:

  src/app/contact/page.tsx
  (search for: "Square Appointments booking button/embed goes here")

CONTACT PAGE STATUS
-------------------

The Contact page now serves two paths:
1. Primary: Square booking CTA ("Book Now")
2. Secondary: WhatsApp chat and question-only enquiry form

Question form delivery is implemented via:
  - Frontend form: /src/components/ContactForm.tsx
  - Backend endpoint: /src/app/api/contact/route.ts
  - Transport: Gmail SMTP using nodemailer

Required environment variables:
  - SMTP_HOST=smtp.gmail.com
  - SMTP_PORT=465
  - SMTP_SECURE=true
  - SMTP_USER=PASTE_GMAIL_ADDRESS_HERE
  - SMTP_PASSWORD=PASTE_GMAIL_APP_PASSWORD_HERE
  - ENQUIRY_TO_EMAIL=PASTE_ENQUIRY_RECEIVING_EMAIL_HERE

Important:
  - Gmail App Password requires Google 2-Step Verification.
  - Use Gmail App Password, not the normal Gmail login password.
  - Do not render business email publicly in customer-facing UI.

ADDING IMAGES
-------------

1. Add images to /public/images/
2. Replace <ImagePlaceholder> components in the relevant page/component
   with Next.js <Image> tags, e.g.:
      import Image from "next/image";
      <Image src="/images/hero.jpg" alt="..." fill className="object-cover" />

CONTACT FORM
------------

Contact form is for enquiries only (not primary booking).
Primary booking should remain Square Appointments CTA links.
Do not route booking requests through the form.
Form submits to /api/contact and sends email via Gmail SMTP.

ADDING A NEW SERVICE
--------------------

1. Add to content/services.json under the correct category
2. Add a matching pricing entry in content/pricing.json using the same serviceId
3. The Services page and pricing page auto-render from the JSON — no code change needed.
4. Optionally add the service to content/bundles.json if it should appear in a bundle.

ADDING A NEW BUNDLE
-------------------

Add an entry to content/bundles.json following the existing pattern.
The bundles page renders all entries in the JSON automatically.

GOING LIVE (VERCEL)
-------------------

1. Push to GitHub
2. Connect repo to Vercel at vercel.com/new
3. Set NEXT_PUBLIC_SITE_URL environment variable to your domain
4. Set NEXT_PUBLIC_SQUARE_BOOKING_URL to your real Square booking URL
5. Set NEXT_PUBLIC_WHATSAPP_CHAT_URL to your real WhatsApp wa.me URL
6. Set SMTP_HOST, SMTP_PORT, SMTP_SECURE, SMTP_USER, SMTP_PASSWORD, and ENQUIRY_TO_EMAIL
7. Ensure Gmail account has 2-Step Verification + App Password enabled
8. Vercel handles build and deployment automatically

LOCAL DEVELOPMENT

ADDING A NEW PAGE

Create a folder under /src/app/ and add a page.tsx file following the
